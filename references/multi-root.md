# Multi-root projects: the shared-core pattern

Read this only when Phase 0 classified the project as **multi-root** — a monorepo with several
packages/apps, or two-plus sibling repos that restate the same facts. Single-root projects
don't need any of this; put cross-cutting facts directly in the root canonical file.

## The problem shared-core solves

In multi-root layouts, the same invariants get copy-pasted: domain rules, schema/enum values,
pricing or fee constants, which package owns what, the agent/skill name tables, anti-hallucination
rules. Each copy drifts independently. shared-core makes there be exactly one copy.

## How it works

1. Put the invariants in **`docs/agents/shared-core.md`** (one file).
2. The root canonical file (`AGENTS.md`) embeds that file verbatim between sentinels:
   ```
   <!-- BEGIN SHARED-CORE hash:XXXX src:docs/agents/shared-core.md — ... -->
   ...generated copy of shared-core.md...
   <!-- END SHARED-CORE -->
   ```
   `agent-docs.sh --sync` regenerates the block and stamps the content hash; `--check`
   recomputes the hash and fails if the embed drifted from the source file. The embed makes the
   invariants visible inline (agents read the canonical file top-to-bottom) while keeping a
   single editable source.
3. Per-directory `AGENTS.md` files in sub-packages do NOT embed shared-core — they just point to
   it ("cross-cutting invariants live in the root `AGENTS.md` SHARED-CORE block /
   `docs/agents/shared-core.md`"). Only the root embeds, so there's one copy per repo.

## Two-plus sibling repos

When invariants must match across *separate* repos (not just packages in one repo):

- Designate ONE repo as **upstream** (author shared-core there).
- Downstream repos pull it during sync by setting `AGENT_DOCS_UPSTREAM`:
  ```bash
  AGENT_DOCS_UPSTREAM=../upstream-repo tools/agent-docs.sh --sync
  ```
  This copies `docs/agents/shared-core.md` from upstream, then embeds it. Match an existing
  "shared layer" sync convention if the project already has one.
- Verify no drift between repos:
  ```bash
  tools/agent-docs.sh --check-cross ../other-repo
  ```
  CI in either repo can't see the other, so this is a local/release-time check plus the visible
  sentinel hash. Document an "edit shared-core upstream, then re-sync downstream" rule.

## What belongs in shared-core vs a directory file

Belongs in shared-core (cross-cutting, drift-prone, referenced for anti-hallucination):
- Domain identity / product one-liner and non-negotiable principles
- Canonical enums, status values, schema/collection names
- Pricing / fee / quota constants
- Repo/package ownership tables; "edit X in place" rules
- Agent and skill name tables (so nobody invokes a name that doesn't exist)
- Anti-hallucination rules (verify-before-generating checklist)

Stays in the directory's own canonical file (specific to that app/package):
- Build/run/test/deploy commands
- Routes, screens, feature structure
- Stack-specific gotchas and patterns
- Directory-local conventions

## shared-core.md starter template

Adapt headings to the project; delete what doesn't apply. Keep it tight — this is reference
ground-truth, not a tutorial.

```markdown
<!-- SHARED-CORE — canonical cross-cutting invariants. Edit ONLY in <upstream repo/path>; -->
<!-- downstream copies are generated by agent-docs.sh. Do not hand-edit embedded copies. -->

# <Project> Shared Core (canonical invariants)

<one-line product description>. <non-negotiable principles, if any>.

## Structure & ownership
| Area | Owned by |
|------|----------|
| <concern> | <package/repo/path> |

## Canonical invariants (anti-hallucination ground truth)
### <enums / status values / schema names>
<the canonical table or list>

### <pricing / quotas / constants>
<the canonical values — "never hardcode other values">

## Anti-hallucination rules (verify before generating)
- <fact category> → <where to verify it>
- Cite file:line for code claims; measure counts from tool output, never estimate.

## Agents / skills (only these names exist)
<tables of the real agent and skill names, so none get invented>
```
